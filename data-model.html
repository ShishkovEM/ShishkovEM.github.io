Модель данных

В настоящем разделе описана модель данных, необходимая для реализации функциональности отображения списка инвестиционных идей.

Для визуализации списка инвестиционных идей используется витрина данных в виде связанных таблиц, хранящихся в базе данных PostrgresSQL.

Ниже представлена предлагаемая структура связанных таблиц.

ER-диаграмма

Следующая ER-диаграмма описывает структуру таблиц, необходимую для отображения списка инвестиционных идей:

Описание сущностей:

  • SECURITY: Это основная таблица, содержащая общие атрибуты всех ценных бумаг, включая их ID, тип (акции, облигации, фонды), название и URL логотипа.

  • STOCK, BOND, FUND: Эти таблицы являются специализированными расширениями таблицы SECURITY и содержат уникальные атрибуты для акций, облигаций и фондов соответственно. Например, для акций это сектор, потенциал доходности и дата включения в топ акций.

  • INVEST_IDEA: Эта таблица содержит информацию об инвестидеях, включая описание и URL для детальной информации. Каждая инвестидея связана с одной или несколькими ценными бумагами.

В следующем разделе представлены SQL-скрипты для создания таблиц, наполнения их тестовыми данными, а также для создания хранимых процедур, релизующих возможности

  • просмотра списка инвестиционных идей

  • просмотра детального описания инвестиционной идеи

  • фильтрации и сортировки отображаемых инвестиционных идей
